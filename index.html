03. Stage 1 Scorer

A dense value scorer starts from $[B, F]$.

Normalization & Scale

Dense float features must be normalized before the model sees them. Float normalization matters more for DNNs than GBDTs because gradients and activations are scale-sensitive.

Architecture Note: Meta retrieval models often use SwishLayerNorm as $y = x \cdot \text{sigmoid}(\text{LayerNorm}(x))$; activation and normalization order is architecture-specific.
Tensors

dense_x: [B, F]  |  hidden: [B, H]  |  score: [B] 

import torch
import torch.nn as nn

class SwishLayerNormBlock(nn.Module):
    def __init__(self, in_dim, out_dim):
        super().__init__()
        self.linear = nn.Linear(in_dim, out_dim)
        self.norm_gate = nn.Sequential(
            nn.LayerNorm(out_dim),
            nn.Sigmoid()
        )
    def forward(self, x): # x: [B, in_dim]
        x = self.linear(x) Shape: [B, out_dim]
        return x * self.norm_gate(x)

class ValueScorer(nn.Module):
    def __init__(self, num_features, hidden_dim):
        super().__init__()
        self.net = nn.Sequential(
            SwishLayerNormBlock(num_features, hidden_dim),
            SwishLayerNormBlock(hidden_dim, hidden_dim),
            nn.Linear(hidden_dim, 1)
        )
    def forward(self, dense_x): dense_x: [B, F]
        return self.net(dense_x).squeeze(-1) Shape: [B]

05. Pairwise Ranking

With session IDs, train preferences within the same context.

Data Generation Difference

Consider a training log with two sessions:

  • Session 1: 4 impressed items, 1 clicked (Item 4), 3 unclicked (Items 1, 2, 3).
  • Session 2: 3 impressed items, 0 clicked (Items 5, 6, 7).

Pointwise Training: Generates 7 feedbacks ($4 + 3$). Every impression is treated as an independent label ($1.0$ or $0.0$). The model spends capacity trying to fit unclicked items in completely inactive sessions (Session 2).

Pairwise Training: Generates 3 pairs ($1 \times 3$ from Session 1). Session 2 has 0 clicks, so it generates **0 pairs** and is ignored. Optimization focuses strictly on ordering inside active sessions.

Context Rule: A positive item from one session is never compared against a negative item from a different session.

06. Pairwise Math

BCEWithLogitsLoss is the logistic ranking loss.

Let $z = \text{score}_i - \text{score}_j$. The model probability that item $i$ should rank above item $j$ is:

$$p(i > j) = \sigma(z) = \frac{1}{1 + \exp(-z)}$$

For a binary target $y$, BCE is:

$$\text{loss} = - y \log(\sigma(z)) - (1 - y) \log(1 - \sigma(z))$$

If $i$ is positive and $j$ is negative, $y = 1$:

$$\text{loss} = -\log(\sigma(\text{score}_{\text{pos}} - \text{score}_{\text{neg}}))$$

Minimizing the loss pushes $\text{score}_{\text{pos}}$ above $\text{score}_{\text{neg}}$. Using logits directly is the numerically stable implementation.

# z = score_pos - score_neg
target = torch.ones_like(z) Shape: [B_pairs]
loss = F.binary_cross_entropy_with_logits(
    z, target
)
07. Feature Interactions

Many ranking signals are naturally "and" conditions.

Conjunction Examples
  • User watched this author and the video is popular.
  • User is price sensitive and the product is discounted.
  • Query intent matches category and seller quality is high.
  • Recent behavior matches the target item, not just any item.
GBDT Intuition

Tree paths naturally encode conjunctions. That is why GBDTs are so strong on dense tabular ranking features.

Scale Limit: The pressure point is scale: thousands of features, learned embeddings, high-cardinality IDs, and sequence features push us toward neural interaction architectures.
08. Neural Interactions

Start by making the cross explicit.

A plain MLP can learn interactions, but a specialized architecture makes the most important cross terms easier to express.

Explicit Cross Pipeline

user_h
+
item_h
+
user_h * item_h
→
Concat [3H]

Note: a sufficiently large MLP can represent these crosses in principle, but explicit interaction layers make useful terms easier to learn with finite data and SGD. 

class ExplicitInteractionScorer(nn.Module):
    def __init__(self, user_dim, item_dim, hidden_dim):
        super().__init__()
        self.user_proj = nn.Linear(user_dim, hidden_dim)
        self.item_proj = nn.Linear(item_dim, hidden_dim)
        self.head = nn.Sequential(
            SwishLayerNormBlock(3 * hidden_dim, hidden_dim),
            nn.Linear(hidden_dim, 1)
        )
    def forward(self, user_x, item_x):
        user_h = self.user_proj(user_x) Shape: [B, H]
        item_h = self.item_proj(item_x) Shape: [B, H]
        cross = user_h * item_h          Cross: [B, H]
        h = torch.cat(
            [user_h, item_h, cross], dim=-1
        ) Concat: [B, 3H]
        return self.head(h).squeeze(-1) Shape: [B]
09. Attention Interactions

Represent features as tokens, then let attention mix them.

The analogy is "features as words." Instead of one flat vector, each feature group can condition on the others through attention.

Feature Groups
→
Project (128d)
→
Self-Attention
→
Mean Pool
→
Score Head

By mapping feature fields (e.g. User History, Category, Time Context) to tokens, the Transformer layer learns non-linear interaction matrices dynamically.

class FeatureTokenRanker(nn.Module):
    def __init__(self, token_sources, dim=128, layers=2):
        super().__init__()
        self.proj = nn.ModuleDict({
            name: nn.Linear(src_dim, dim)
            for name, src_dim in token_sources.items()
        })
        enc_layer = nn.TransformerEncoderLayer(
            d_model=dim, nhead=4,
            dim_feedforward=4 * dim, batch_first=True
        )
        self.encoder = nn.TransformerEncoder(enc_layer, layers)
        self.head = nn.Linear(dim, 1)
    def forward(self, feature_groups):
        tokens = [self.proj[name](feature_groups[name]) 
                  for name in self.proj]
        x = torch.stack(tokens, dim=1)  Shape: [B, T, 128]
        x = self.encoder(x)             Shape: [B, T, 128]
        pooled = x.mean(dim=1)          Pooled: [B, 128]
        return self.head(pooled).squeeze(-1)
10. Multi-Task

One shared representation can feed multiple task heads.

A value model often combines several predicted outcomes into the final ranking score: click, conversion, dwell, purchase value, or other business terms.

dense_x: [B, F]
→
Backbone h: [B, H]
→
Task Heads: [B, T]
Serving Value Formula:
$\text{vm\_score} = \sum_{t} P(\text{Task}_t) \times \text{Value}_t$ 
class MultiTaskValueModel(nn.Module):
    def __init__(self, num_features, hidden_dim, num_tasks):
        super().__init__()
        self.backbone = nn.Sequential(
            SwishLayerNormBlock(num_features, hidden_dim),
            SwishLayerNormBlock(hidden_dim, hidden_dim)
        )
        self.heads = nn.Linear(hidden_dim, num_tasks)
    def forward(self, dense_x):
        h = self.backbone(dense_x)      Shared: [B, H]
        return self.heads(h)            Heads: [B, T]

# At serving:
logits = model(dense_x)
probs = torch.sigmoid(logits) Probs: [B, T]
vm_score = (probs * task_values).sum(dim=-1) Score: [B]
11. Global MMoE

Global MMoE learns task-choice routing with row-stable weights.

Each task learns its own $[E]$-dimensional distribution over shared experts. The weights are learned, but they are global for that task: they do not vary by row (example).

Variable Dimensions
  • • experts: [B, E, D]
  • • weights: [T, E]
  • • task_repr: [B, T, D]

By soft-averaging the experts globally, tasks share representation space while specializing their heads, using torch.einsum for fast batch tensor contraction. 

class GlobalMMoE(nn.Module):
    def __init__(self, in_dim, expert_dim, num_experts, num_tasks):
        super().__init__()
        self.experts = nn.ModuleList([
            nn.Linear(in_dim, expert_dim)
            for _ in range(num_experts)
        ])
        self.task_expert_logits = nn.Parameter(
            torch.zeros(num_tasks, num_experts)
        )
    def forward(self, h):
        expert_out = torch.stack(
            [expert(h) for expert in self.experts], dim=1
        ) Experts: [B, E, D]
        weights = torch.softmax(
            self.task_expert_logits, dim=-1
        ) Weights: [T, E]
        return torch.einsum(
            "bed, te->btd", expert_out, weights
        ) Routing: [B, T, D]

13. Dynamic MMoE

Row-conditioned MMoE makes the expert mixture example-specific.

Global MMoE learns one $[T, E]$ routing matrix. Row-conditioned MMoE learns gates shaped $[B, T, E]$, so the same task can route different examples through different expert mixtures.

Conditional Gating:
Instead of static parameters, the gate weights are computed dynamically: $\text{gate\_logits} = \text{Linear}(h)$ for each task.

experts: [B, E, D] | gates: [B, T, E] | task_repr: [B, T, D] 

class MMoE(nn.Module):
    def __init__(self, in_dim, expert_dim, num_experts, num_tasks):
        super().__init__()
        self.experts = nn.ModuleList([
            nn.Linear(in_dim, expert_dim)
            for _ in range(num_experts)
        ])
        self.gates = nn.ModuleList([
            nn.Linear(in_dim, num_experts)
            for _ in range(num_tasks)
        ])
    def forward(self, h):
        expert_out = torch.stack(
            [expert(h) for expert in self.experts], dim=1
        ) Experts: [B, E, D]
        gate_logits = torch.stack(
            [gate(h) for gate in self.gates], dim=1
        ) Gates: [B, T, E]
        gate_weights = torch.softmax(gate_logits, dim=-1)
        return torch.einsum(
            "bed, bte->btd", expert_out, gate_weights
        ) Task reps: [B, T, D]
14. Feature Embeddings

Domain adapters project embeddings before concatenation.

"Domain adaptation" here can be explained simply: project a feature from its original representation into the ranking model's hidden space before concatenating.

Concatenation Flow

Dense [B, F] → MLP → dense_h [B, H]
Embedding [B, D_emb] → Adapter → emb_h [B, H]
Concat → [B, 2H] → Scoring Head 
class DensePlusEmbeddingRanker(nn.Module):
    def __init__(self, dense_dim, emb_dim, hidden_dim):
        super().__init__()
        self.dense_mlp = nn.Sequential(
            SwishLayerNormBlock(dense_dim, hidden_dim),
            SwishLayerNormBlock(hidden_dim, hidden_dim)
        )
        self.emb_adapter = nn.Sequential(
            nn.Linear(emb_dim, hidden_dim),
            nn.LayerNorm(hidden_dim),
            nn.SiLU()
        )
        self.head = nn.Linear(2 * hidden_dim, 1)
    def forward(self, dense_x, emb):
        dense_h = self.dense_mlp(dense_x) Dense: [B, H]
        emb_h = self.emb_adapter(emb)     Embedding: [B, H]
        h = torch.cat([dense_h, emb_h], dim=-1) Concat: [B, 2H]
        return self.head(h).squeeze(-1)
15. Sequences

The first useful sequence feature is often masked mean pooling.

This is cheap, stable, and surprisingly competitive. It asks: "What has this user generally done?"

Pooling Dimensions

seq: [B, N, D]  |  mask: [B, N]  |  pooled: [B, D]

The boolean sequence mask prevents padding tokens (unrealized history steps) from bleeding into the mean representation, clamping the denominator min to 1.0 to avoid division by zero. 

def masked_mean(seq, mask):
    # seq: [B, N, D] | mask: [B, N]
    mask_f = mask.float()
    denom = mask_f.sum(dim=1).clamp_min(1.0) Shape: [B]
    total = (seq * mask_f[:, :, None]).sum(dim=1) Sum: [B, D]
    return total / denom[:, None] Mean: [B, D]

# In model forward:
history_h = masked_mean(history_seq, history_mask)
history_h = history_adapter(history_h)
h = torch.cat(
    [dense_h, emb_h, history_h], dim=-1
)
16. Time-Sliced Pooling

A strong practical extension: pool history by recency bucket.

Mean pooling erases time. Time-sliced pooling preserves a small amount of recency structure while keeping the model simple.

Recency Buckets

Recent: last 24 hours
Medium: last week (24h - 7d)
Old: older history (> 7d)

Pooling each bucket separately and concatenating them creates a 3D representation: [B, 3D], letting the dense layers learn weight decays over time. 

# seq: [B, N, D] | age_hours: [B, N] | seq_mask: [B, N]
recent_mask = seq_mask & (age_hours < 24)
mid_mask = (seq_mask & (age_hours >= 24) 
            & (age_hours < 24 * 7))
old_mask = seq_mask & (age_hours >= 24 * 7)

recent_pool = masked_mean(seq, recent_mask) Recent: [B, D]
mid_pool = masked_mean(seq, mid_mask)       Medium: [B, D]
old_pool = masked_mean(seq, old_mask)       Old: [B, D]

time_sliced = torch.cat(
    [recent_pool, mid_pool, old_pool], dim=-1
) Concat: [B, 3D]
17. Target Attention

Now ask which history events matter for this target.

  • Mean Pooling asks what the user generally did.
  • Time-Sliced Pooling asks when they did it.
  • Target Attention asks whether each history event is relevant to the candidate being ranked now.
$$\text{logit}[b, n] = \frac{\text{query}(\text{target}[b]) \cdot \text{key}(\text{seq}[b, n])}{\sqrt{D}}$$

18. Target Attention Code

The definitional PyTorch version.

The dot product says how much each history event matches the current target candidate. Valid history items are kept, while padding steps are masked to negative infinity before softmax.

Tensor Shapes
  • • target: [B, D]
  • • seq: [B, N, D]
  • • seq_mask: [B, N]
  • • logits: [B, N]
import math

class TargetAttention(nn.Module):
    def __init__(self, dim):
        super().__init__()
        self.q_proj = nn.Linear(dim, dim)
        self.k_proj = nn.Linear(dim, dim)
        self.v_proj = nn.Linear(dim, dim)
        
    def logits(self, target, seq, seq_mask):
        q = self.q_proj(target)[:, None, :] Query: [B, 1, D]
        k = self.k_proj(seq)               Keys: [B, N, D]
        logits = (q * k).sum(dim=-1) / math.sqrt(k.size(-1)) Logits: [B, N]
        return logits.masked_fill(~seq_mask, -1e9)
19. Attention Gating

Softmax chooses among history; sigmoid can reject all history.

Softmax Attention

Distributes probability mass across valid history events. Good when at least one event should explain the target (forces competition).

weights = torch.softmax(logits, dim=-1)
weights = weights * seq_mask.float()
weights = weights / weights.sum(
    dim=-1, keepdim=True
).clamp_min(1e-6) Weights: [B, N]
context = (weights[:, :, None] * v).sum(dim=1)
Sigmoid Attention

Gates each event independently. Useful when every history event may be irrelevant to the target (allows zeroing out representation).

gates = torch.sigmoid(logits) Gates: [B, N]
gates = gates * seq_mask.float()
context = (gates[:, :, None] * v).sum(dim=1)
gate_strength = gates.sum(
    dim=-1, keepdim=True
)
20. Full Architecture

A modern ranking model layers these ideas.

Model Piping Flow

• Dense Features + Embeddings → Adapters → Concat
• History Sequence → Pooling / Attention → Concat
• All Features → Interaction Layers → Dense Tower
• Tower Output → Multi-Task Experts / Heads → VM Score

By organizing dense, sparse, and historical parameters in this structured funnel, the model combines wide-and-deep learning with multi-task gating to predict click and value outcomes.

Scoring: Ranking starts with a scalar score per candidate. Everything else exists to make that score better.
Objectives: Pointwise losses fit labels. Pairwise losses directly teach ordering inside a session or request context.
Architecture: Interactions, experts, embeddings, and sequence attention add useful inductive bias for ranking data.
